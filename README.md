# Credit Limit CRM (Тестовое задание)

## Описание проекта

Это прототип внутреннего модуля CRM для управления кредитными лимитами клиентов банка.  
Основная цель — просмотр списка заявок на изменение кредитного лимита, их редактирование и сохранение истории действий пользователя за сессию.

**Функционал:**
- Таблица с заявками клиентов (ID, ФИО, текущий и запрошенный лимит, статус)
- Фильтрация по статусу заявки (New / Approved / Rejected)
- Поиск по ФИО с использованием debounce
- Редактирование заявки через модальное окно
    - Маскирование счета клиента
    - Поля «Новый лимит» и «Причина изменения» с зависимой логикой
    - Валидация лимита (0–10 000 000 ₽)
    - Если лимит >1 000 000 ₽, добавляется причина «Особый риск» и поле становится обязательным
- Лог сессии: хранение всех действий пользователя за текущую сессию (изменение лимита, смена статуса)

---

## Стек технологий

- **React + TypeScript** — фронтенд
- **Redux Toolkit** — управление состоянием приложения
- **Custom Hooks** — вынос бизнес-логики из компонентов
- **Mock API** — имитация асинхронных запросов через `Promise + setTimeout`
- Чистая архитектура с разделением слоёв:
    - `entities` — доменные сущности (клиенты, типы, API)
    - `features` — кастомные хуки для логики поиска, фильтрации, редактирования
    - `widgets` — UI-компоненты (таблица, модалка, лог)
    - `shared` — утилиты и хелперы (маскирование счета, валидация, логирование)

## Запуск проекта

1. Клонировать репозиторий:

```bash
git clone https://github.com/NurlanuuluJainak/Credit-Limit-CRM.git
cd credit-limit-crm
```

Установить зависимости:
```
npm install

```


Запустить в режиме разработки:
```
npm run dev
```


Проект будет доступен по адресу http://localhost:5173

Сборка для продакшена:
```
npm run build
```


Архитектура и решения

Redux Toolkit используется для централизованного хранения данных клиентов и логов сессии.

Custom Hooks (useClientLogic, useDebounce) обеспечивают чистоту компонентов и вынос бизнес-логики.

Валидация лимита и логика «Особый риск» вынесена в утилиты (shared/utils.ts) для повторного использования.

Сессионный лог хранится в Redux и обновляется при каждом действии пользователя.

Маскирование счета через утилиту maskAccount гарантирует безопасность отображения данных.

Debounce реализован для поиска по ФИО, чтобы оптимизировать рендер и вызовы API.
